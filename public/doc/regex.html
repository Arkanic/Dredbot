Regex for ;ldb

The ;ldb leaderboard command supports regex whilst using the -r tag.

Example:

;ldb -r /^noob/i

Basic regex is formatted with two forwards slashes, with the search query going between the slashes and the settings at the end.

Basics

Anchors

^Start. - matches any string that starts with "Start." End$ - matches any string that ends with "End."

Quantifiers

123* - matches any string that has a 12 followed by zero or more of the character 3. 123+ - matches any string that has 12 followed by one or more of the character 3. 123? - matches any string that has 0 or 1 of the character 3. 123{2} - matches any string that has 12 followed by 2 3s. 123{2,} - matches any string that has 12 followed by 2 or more 3s. 123{2,5} - matches any string that has 12 followed by 2 to 5 3s. 1(23)* - matches any string that has 1 followed by any amount of the sequence "23". 1(23){2,5} - matches any string that has 1 followed by 2 to 5 of the sequence "23".

OR Operators

1(2|3) - matches any string that has a 1 followed by a 2 or 3, and captures 2 or 3. 1[23] - same as above, but does not capture 2 or 3.

Character matches

\d - matches any single character digit. \w - matches any word character (alphanumeric + underscore). \s - matches any whitespace character (space, tab, 0 width, etc). . - matches anything.

You can capitalise the characters in the backspace to make them match the opposite, e.g. \D would match any non-digit, and \S would match any non-whitespace character.

In order to be taken literally, you must use the escape characters ^.[$()|*+?{\ with a backslash as they have a special meaning.

\$\d - matches a string that has a $ before one digit.

You can also match non-printables like newlines, tabs, and returns (although impossible to name a ship with the latter) with their respective backslash identifiers (\n, \t, and \r.)

Flags

regex is normally comes with the form /abc/, where the search pattern is identified by being inbetween the two slashes (/). At the end we can specify specific flags that will affect how the search comes out.

  • g (global) does not return after the first match, restarting the subsequent searches from the end of the previous match.
  • m (multi-line) [useless for ;ldb search] when enabled ^ and $ will match the start and end of a line, rather than the whole string.
  • i (insensitive) makes the entire string case-insensitive.

Brackets

[abc] - matches any string that has either an a or a b or a c. same as (a|b|c). [a-c] - same as previous. [a-fA-F0-9]- matches any single hexadecimal character. [0-9]h - matches any number before a h. [^a-zA-Z] - matches any character that is not a lower or upper case alphabetic character.
